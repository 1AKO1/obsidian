**推荐**
1. 根据业务需要自行评估核心线程数、最大线程数、线程回收策略、队列类型、拒绝策略
2. 线程命名要有实际含义，方便排查
3. 增加监控，例如线程队列的积压或者线程数量的膨胀，线程池一般情况下除非拒绝了，否则啥动静都没有。

### 可以适当调整线程池的默认行为
**线程池默认工作行为**
- corePoolSize个核心线程不是刚开始就创建好了，是懒加载。
- 核心线程满了会先入队列，等队列满了再开始扩容线程池，直到maximumPoolSize为止
- 如果队列满了 且 线程池也最大了，就按照拒绝策略处理。
- 当线程数大于核心线程数时，非核心线程等待keepAliveTime后仍然空闲，就会回收。

**一些操作**
- 声明线程池后立即调用prestartAllCoreThreads方法，来启动所有核心线程；
- 传入true给allowCoreThreadTimeOut方法，来让线程池在空闲的时候同样回收核心线程。

### 确认线程池是否可复用
**线程池首先要是必须要复用的**：要不然每次获取线程池的时候都会创建一个新的。如果任务时间比较长，而且keepAliveTime也长的话，那内存就爆了

**不同逻辑的线程池最好不要混用**：计算型任务和IO型任务对于线程的配置需要不同，所以最好不要混用。

**有依赖关系的父子线程不要共用同一个线程池**：父线程的执行依赖子线程，如果父线程把线程池占满了，子线程无法申请，那父线程也不会结束，直接死锁了。
